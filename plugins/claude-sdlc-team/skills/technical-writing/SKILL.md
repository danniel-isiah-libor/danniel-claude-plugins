# Technical Writing

## Principles
- Write for a specific audience and their goal; lead with what/why before how.
- Show, don't just tell: runnable examples with expected output.
- Structure: overview → prerequisites → steps → reference → troubleshooting.
- One idea per paragraph; active voice; define each term once.

## Doc types
- **README**: what it is, install, quickstart, key commands, links.
- **API docs**: endpoint/signature, params, returns, errors, example.
- **Changelog**: see `release-management`.

## Output (06-docs.md + doc files)
List the docs created/updated and why; keep docs beside the code they describe.
