# UI/UX Design

## Principles
- UX first: one clear primary action per screen; minimize steps; immediate feedback; forgiving (undo, confirm destructive actions).
- Visual hierarchy: use size/weight/spacing/contrast to guide the eye; a consistent spacing scale.
- Accessibility (WCAG): semantic HTML, labels, visible focus states, keyboard nav, 4.5:1 text contrast, hit targets ≥ 44px.
- Responsive & cross-platform: mobile-first; fluid layouts; respect platform conventions (web/mobile/desktop); test breakpoints.

## Tailwind as a design system
Centralize tokens in the theme (colors, spacing, type scale); use utilities consistently; extract repeated patterns into components; drive dark mode from tokens, not ad-hoc classes.

## Output
For design: a UX flow + component/layout notes. For build: an accessible, responsive, token-driven UI.
